# Структура проекта ComToAir

```
ComToAir/
│
├── platformio.ini                    # Конфигурация PlatformIO и ESP-IDF
├── README.md                         # Основное описание проекта
├── TECHNICAL_SPECIFICATION.md        # Техническое задание
├── BUILD_INSTRUCTIONS.md             # Инструкция по сборке
├── PROJECT_STRUCTURE.md              # Этот файл
├── .gitignore                        # Игнорируемые файлы для Git
│
├── src/                              # Исходный код
│   └── main.cpp                      # Главный файл приложения
│
├── include/                          # Заголовочные файлы
│   ├── config.h                      # Конфигурационные параметры
│   ├── rs232_handler.h               # Интерфейс обработчика RS-232
│   ├── wifi_manager.h                # Интерфейс управления WiFi
│   └── web_server.h                  # Интерфейс веб-сервера
│
├── data/                             # Статические файлы для веб-интерфейса
│   └── index.html                    # Главная страница веб-интерфейса
│
└── test/                             # Тесты (будущее расширение)
    └── (тесты будут добавлены позже)
```

## Описание файлов

### Корневые файлы

- **platformio.ini** - Конфигурационный файл PlatformIO, определяет:
  - Целевую платформу (XIAO ESP32-C6)
  - Framework (ESP-IDF)
  - Параметры сборки и загрузки
  - Настройки монитора последовательного порта

- **README.md** - Краткое описание проекта, основные возможности, быстрый старт

- **TECHNICAL_SPECIFICATION.md** - Полное техническое задание с:
  - Общими сведениями о проекте
  - Техническими требованиями
  - Функциональными требованиями
  - Этапами разработки
  - Критериями приемки

- **BUILD_INSTRUCTIONS.md** - Подробная инструкция по:
  - Установке среды разработки
  - Аппаратному подключению
  - Компиляции и загрузке
  - Настройке и тестированию

- **.gitignore** - Список файлов и папок, игнорируемых Git

### Исходный код (src/)

- **main.cpp** - Главный файл приложения, содержащий:
  - Инициализацию системы
  - Настройку UART для RS-232
  - Настройку WiFi
  - Запуск веб-сервера
  - Основной цикл обработки данных

### Заголовочные файлы (include/)

- **config.h** - Централизованные конфигурационные параметры:
  - Параметры UART
  - Настройки WiFi
  - Параметры веб-сервера
  - Размеры буферов
  - Таймауты

- **rs232_handler.h** - Интерфейс модуля работы с RS-232:
  - Инициализация UART
  - Чтение/запись данных
  - Настройка параметров связи
  - Управление буферами

- **wifi_manager.h** - Интерфейс модуля управления WiFi:
  - Подключение к сети
  - Режим точки доступа
  - Получение статуса
  - Управление конфигурацией

- **web_server.h** - Интерфейс веб-сервера:
  - Запуск/остановка сервера
  - Обработка HTTP запросов
  - API endpoints
  - Управление данными

### Статические файлы (data/)

- **index.html** - Веб-интерфейс для мониторинга и управления:
  - Отображение статуса устройства
  - Мониторинг данных RS-232 в реальном времени
  - Настройка параметров
  - Современный адаптивный дизайн

### Тесты (test/)

Директория для модульных и интеграционных тестов (будет заполнена в процессе разработки)

## Расширение проекта

При добавлении новых функций рекомендуется:

1. **Новые модули** - создавать в `include/` и `src/` с соответствующими именами
2. **Конфигурация** - добавлять в `include/config.h`
3. **Веб-ресурсы** - размещать в `data/`
4. **Документация** - обновлять соответствующие .md файлы

## Соглашения о кодировании

- Имена файлов: snake_case (например, `rs232_handler.h`)
- Имена функций: snake_case (например, `rs232_init()`)
- Имена констант: UPPER_CASE (например, `UART_BUF_SIZE`)
- Комментарии: на русском языке для пользовательской документации, на английском для технических комментариев в коде
